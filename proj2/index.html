Overview

In this project, there were two main parts. In the first part, I explore bezier curves and surfaces, allowing me to model a smooth and infinitely scalable surface or curve. Part 2: Bezier surfaces with separable 1D de Casteljau subdivision

Next, we extend Casteljau's algorithm to Bezier surfaces, allowing us to create smooth surfaces rather than those defined by triangular meshes. We start with a grid of n x n control points. We first evaluate the Bezier point along the rows, so that we have n control points, one for each row. This is done by treating each row as a vector of control points, and applying the recursive algorithm from above for some parameter u. After receiving an n-length vector from this operation, we repeat the same iterative de Casteljau algorithm using the column parameter v, again getting a single point at the end, which will lie on the Bezier surface. Section II: Sampling

In this next section is where geometry and visualization really came in handy to visualize and manipulate mesh pointers. I worked closely with the HalfedgeMesh class that was provided, implemting a data structure consisting of half edges, vertices, edges, and faces. Unike a simple list of vertices and triangles, using the HalfedgeMesh allows for meaningful traversal and manipulation of the mesh, such as flipping edges, splitting triangles, and upsampling the mesh to be more detailed. In the data structure, half edges act as the "glue," storing connectivity information to each of the other elements. The other elements simply have pointers to one of their half-edges.

Part 3: Average normals for half-edge meshes

I started with implementing area-weighted normal vectors at the vertices that could then be used for Phong shading. Instead of giving each triangle in the mesh a single color (flat shading), these vertex normals could then be used to interpolate colors across triangles in the mesh, offering a much smoother color, in a technique known as Phong shading. When implementing the average normal algorithm, the goal was to find the normals for each of the non-boundary faces surrounding a vertex and then perform a weighted average based on their magnitudes. To make this operation simpler, I traversed through all neighoring vertices, creating a two vectors that go between the origin vertex and two neighboring vertices. These vectors trace the sides of an adjacent face to the vertex, so taking their cross product gives me the normal vector of that triangular face. It also provides the magnitude, which allows me to weight that normal accordingly. As I iterated through all neighboring vertices, I took successive cross products of the described vectors and kept them in a running sum. Then, I returned the unit vector in the direction of that weighted sum, to represent the average normal direction of the vector's surrounding faces.

Part 4: Half-edge flip

In this next section, I implemented edge flips. Edge flips don't require any deletion of elements, so it simply requires re-assigning pointers. Being my first time working with meshes, I struggled trying to keep track of which pointers to change. Since every element is pointing to a half-edge, moving a half-edge around felt like a daunting tasks. What was extremely helpful was drawing out a diagram of all possible elements that could be impacted by an edge flip-- all inner and outer edges and halfedges, all vertices, and the two faces. From there, I also created pointers to these elements in code, starting with the halfedge of the edge that I was trying to flip. My initial code was long, and buggy, so after getting a grasp of how to create pointers to surrounding elements, I did another re-assessment. This time I only created pointers to elements whose pointers may need to be re-assigned. For edge flips, outer halfedges are not impacted, so I removed those from my code. I also changed by naming system to have a triangle A and B, which made tracking elements much easier throughout the next few tasks.

Part 5: Half-edge split

Next, I implemented the edge split operation. After performing the edge flip and going into excruciating debugging detail, I approached the edge split problem much more methodically. I labelled the triangles before as A and B, and called the additional faces C and D. The halfedges around these triangles were given names according the triangle they were in. This made following pointers a lot easier. When an edge was split, I first found the midpoint of its two vertices through its halfedges, and created a new vertex at that position. From there, I created 2 additional faces, 3 additional edges, and 6 additional half-edges in order to properly annotate the new triangles created by the edge split. Beyond this, my split edge did not need too much additional debugging, although to implement upsampling in the next step, I did come back to flag which edges are newly created and which are part of the original mesh.

Part 6: Loop subdivision for mesh upsampling

Finally, I made use of my flipEdge and splitEdge functions to implement a more complication mesh manipulation: upsampling using loop subdivision. This takes a coarse polygon mesg and converts it into a high resolution one, making the mesh smoother through interpolatation as it does so. First, each vertex is assigned a new position, computed as a weighted average of its neighbors depending on its vertex degree. Then, for each edge, I compute the location of where a new vertex would be when I split that edge, again as a weighted average of the vertices of the edge's two surrounding faces. From there, I go through and split every edge in the mesh. Then, to ensure even-looking triangles, I flip any edge that goes between an old vertex and a new vertex. The end result is a smoother mesh of higher resolution.

What I notice from subdividing some meshes is that we cannot get a perfectly smooth division-- instead of becoming a sphere, an initial cube becomes a round blob that has pointier regions. It is an assymetric shape. Below, I show subdivision steps of the cube. The cube also illustrates how sharp corners no longer stay sharp, and edges are also divides and smoothed out.


In an effort to maintain a cube-like shape, I tried pre-splitting all the diagonal edges on the cube's faces. This helped the cube subdivide much more symmetrically, because the triangles in the starting cube were then symmetric about the cube's center. Even after several stages of subdivision, this cube is able to maintain a smooth but cube-like shape.


Then, to try to keep sharp corners intact, I tried flipping diagonals on the cubes faces so that vertices had higher degree to begin with, as shown below. This helped maintain some more pointy corners, this time in a triangula shape.


These unintended shapes result from using a generalized averaging method for vertex positions after upsampling. As a result, vertices are pulled closer together, so even when edges are meant to be maintained, we get a smoother shape as a result. Intentional maintaining symmetry in the object helps alleviate these assymetrical effects, and sometimes collapsing round faces though a series of edges and flips can create sharper edged objects.
