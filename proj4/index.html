In this final project, I explored simulating fabric using a mass and spring model. The cloth was created using point masses in a grid, and springs were added between them to enforce structural, shearing, and bending constraints. From there, each point mass would be affected by external and spring correction forces at each timestep, causing the cloth to fall from air and interact with itself and other objects, including spheres and planes. After the movement simulation was complete, I added a set of shaders that allowed the cloth to take on various textures, including diffuse shading, bling-phong shading, texture maps, displacement maps, and environment mirrors. The most challenging part of this lab was understanding the simplifications we made to the cloth model, and why discrepancies still existed from a realistic cloth. For example, after we implemented self-collisions, our cloth would continue moving even after we would assume it should have reached a resting state.

The most interesting part of this project was implementing all the different shaders. I’ve never used GLSL before, so this was a great quick introduction to shaders!

Part 1: Masses and springs

In part 1, I build the cloth by adding point masses to the cloth’s existing point-mass vector in row-major order, so the total number of masses is equal to the cloth’s width times the cloth’s height. For horizontal cloths, the point masses vary across the xz plane with a y-coordinate of 1.0, and for a vertical cloth, the point masses vary across the xy plane while the z-coordinate varies slightly. Certain point masses are also pinned, such that if the cloth falls according to the forces acting upon it, these point masses will remain stationary. From there, I also added spring constraints, which include structural, shearing, and bending constraints. Part 2: Simulation via numerical integration

In the next part, I implemented a simulation of the forces acting upon the cloth, which include external forces (gravity) and spring correction forces. The external forces are calculated using F = ma, and then the spring correction force is calculated using Hooke’s law for all the spring types that are enabled. In the GUI, we can toggle which spring types are enabled. From there, we can calculate the new positions of the masses, which is approximated using Verlet integration. Instead of using the velocity, we compute the new position using the current and previous positions and the acceleration. If a point mass is pinned, it doesn’t move.

From there, we want to make sure that point masses haven’t strayed too far from each other to maintain the shape of the cloth, so we ensure that all springs are still within 10% of their resting positions. If they have extended beyond this, I correct their positions by modifying the unpinned point masses at their endpoints.

I then played around with testing different values for ks (spring constant), density, and damping.

I noticed that a smaller spring constant allowed the cloth to be more flexible with smaller wrinkles, bouncing for a while. With a higher spring constant, the cloth was much stiffer, and had larger wrinkles.

Part 3: Handling collisions with other objects

Next, I handled collisions with primitives, so that the cloth could rest on a sphere and on a plane. This required updating the cloth’s simulation function to check for collisions with all the objects in the scene.

For the sphere, I detected if a point mass entered the primitive by checking if the distance from the center was less than the sphere’s radius. Next, I found a tangent vector starting from the sphere’s origin, on the in the same direction as the point mass’s position. I create a correction vector to push the point mass in this direction using its last position so it is no longer inside the sphere. The correction term is also scaled to account for friction.

The correction for a plane is very similar, but handles bidirectional cases for when a point mass crosses from one side of the plane to the other. We can figure this out by checking if the point mass’s position dotted with the plane’s normal vector has changed sign. From there,  I again create a correction vector that is applied to the last position, and also offset some so that the cloth isn’t lying exactly on the plane.

One challenge I encountered here was that I wasn’t properly handling when a point mass’s position was exactly on the plane (dot product was 0), so for a while, some of the cloth was slipping through the plane.

Changing the spring constant here affects how loosely the cloth hangs. With a bigger spring constant, the cloth is stiffer, so it doesn’t fall as much and keeps a stiffer shape.

Part 4: Handling self-collisions

In the next step, I implemented the ability for the cloth to collide with itself, rather than slip into itself as it had been doing previously. I first updated the cloth’s simulation function to check for self-collisions. To speed this check up, we only check the distance of the point mass to a few other point masses within a 3D box near it. We create a hash function to determine which point masses to check– each point mass is hashed to a 3D box and stored in a vector of point masses. Then, in the self-collide function, if a point mass gets too close to a candidate point mass (< 2 * cloth thickness away), we compute a correction vector to repel it away. We sum up the correction vectors for all candidate point masses in the 3D box that are too close, average them out, scale by the number simulation_steps, and then reposition the point mass.
